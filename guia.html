Guía Completa y Detallada

Acciones Principales y Pagos Rápidos

Estos atajos funcionan en cualquier parte de la pantalla de ventas para acciones inmediatas.

  • F1 - Finaliza la venta por el total exacto en Contado.
  • F2 - Finaliza la venta por el total exacto en Transferencia.
  • F3 - Finaliza la venta por el total exacto en Débito.
  • F4 - Finaliza la venta por el total exacto en Crédito (con recargo).
La Tecla `Escape` - El Asistente Universal

La tecla ESC actúa como un "paso atrás" inteligente, con un comportamiento de 3 niveles:

1er Toque

Si no estás en el buscador, te lleva directamente al campo "Buscar Productos".

2do Toque

Si estás en el buscador y hay texto, limpia el texto y los resultados.

3er Toque

Si el buscador ya está vacío, te pregunta si deseas limpiar todo el ticket.

Búsqueda y Edición del Ticket

Atajos para el flujo principal: buscar, agregar y modificar productos.

  • En el Buscador: Usa ↓ / ↑ para navegar la lista de resultados y Enter para agregar el producto seleccionado. Funciona instantáneamente con códigos de barras.
  • Doble `Enter` Rápido: Presiona Enter dos veces seguidas en cualquier lugar para saltar a editar la cantidad del último producto añadido.
  • En un campo de Cantidad: Usa ↓ / ↑ para saltar entre los campos de cantidad de los otros productos del ticket.
  • En un campo de Cantidad: Presiona Enter para finalizar la edición y volver al buscador.

Guía de Pase a Producción

Preparando tu POS 2025 para el Entorno Real

Estos son los pasos cruciales para desplegar tu sistema POS 2025 a un entorno de producción, asegurando que se conecte con tus servicios reales (Tiendanube, ARCA, Firebase de Producción).

  1. 1. Conexión Automática a Base de Datos de Producción (Firebase)
    Tu sistema ya está configurado para cambiar automáticamente al entorno de producción de Firebase.
    No necesitas hacer nada con el código firebase.js. Cuando subas los archivos a tu servidor web (por ejemplo, Firebase Hosting, Vercel, etc.) y dejes de usar localhost o 127.0.0.1, el sistema detectará automáticamente que está en producción y se conectará a la base de datos cajadiaria-infotech.
  2. 2. Despliegue de Cloud Functions a Producción
    Las "Cloud Functions" (los robots que sincronizan con Tiendanube y generan backups) deben ser desplegadas en tu proyecto de Firebase de producción.
    Abre tu terminal, navega hasta la carpeta POS 2025 y ejecuta los siguientes comandos: 
    firebase use cajadiaria-infotech
cd functions
firebase deploy --only functions

    Al finalizar, la consola de Firebase te mostrará la URL del webhookTiendanube. ¡Cópiala, la necesitarás en el paso 4!
  3. 3. Configuración de Tiendanube y ARCA (Desde tu POS 2025)
    Una vez que tu POS 2025 esté corriendo en el entorno de producción (o en el stage, si quieres probar primero), debes ingresar las credenciales correctas.
    Accede a tu POS 2025 (desde tu dominio o tu URL de stage), ve a la sección "Configuración" y completa los siguientes datos:
    • Integración Tiendanube:
      • URL Base de la Tienda: La URL completa de tu tienda (ej: https://mitienda.mitiendanube.com).
      • ID de la Tienda (User ID) y Token: Obtenidos mediante la instalación de tu App privada.
      ¿Cómo obtener el Token y el ID ahora?
      1. Ingresa al Portal de Socios de Tiendanube y ten a mano el App ID (Client ID) y Client Secret de tu aplicación.
      2. Inicia sesión en la tienda de Tiendanube de producción.
      3. En otra pestaña, abre la URL:
        https://www.tiendanube.com/apps/TU_APP_ID/authorize
      4. Acepta los permisos. Serás redirigido a una página. Mira la barra de direcciones y copia el valor de ?code=...
        ¡OJO! Este código caduca en minutos y sirve UNA SOLA VEZ. Úsalo rápido.
      5. Para obtener el token, puedes usar la Consola del Navegador (F12 > Pestaña Console) pegando este código (JavaScript): 
        fetch("https://www.tiendanube.com/apps/authorize/token", {
  headers: { "Content-Type": "application/x-www-form-urlencoded" },
  body: "client_id=TU_APP_ID&client_secret=TU_CLIENT_SECRET&grant_type=authorization_code&code=EL_CODE",
  method: "POST"
}).then(r => r.json()).then(console.log);
        O si prefieres usar una terminal (como Cloud Shell o Bash), usa este comando (cURL): 
        curl -X POST "https://www.tiendanube.com/apps/authorize/token" \
-H "Content-Type: application/x-www-form-urlencoded" \
-d "client_id=TU_APP_ID&client_secret=TU_CLIENT_SECRET&grant_type=authorization_code&code=EL_CODE"
      6. La respuesta será un texto/JSON. Copia el access_token (Token) y el user_id (ID de Tienda) y pégalos en la Configuración del POS.
      ¿Te da error "invalid_grant"?
      Significa que el código (code) expiró, escribiste mal un dato o ya lo usaste. Repite desde el paso 3 para generar uno nuevo y canjéalo inmediatamente.
    • Facturación Automática ARCA:
      • URL Base de la API: La URL de tu servicio ARCA (ej: https://tu-api-arca.com).
      • CUIT Emisor: Tu CUIT.
      • API Key: Tu llave secreta para ARCA.
      • Importante: Asegúrate de marcar "Entorno de Producción (Servidores Reales)".
      • Configura los métodos de pago para auto-facturar según tu preferencia.

    No olvides hacer clic en "Guardar" en cada sección después de completar los datos.
  4. 4. Conectar Tiendanube con tu POS (Configuración de Webhooks)
    Debes decirle a tu tienda de Tiendanube que avise a tu POS 2025 cuando haya un nuevo evento. Al usar una app privada, los webhooks se registran mediante la API.
    Abre tu Cloud Shell (o cualquier terminal) y ejecuta estos dos comandos. Reemplaza TU_ID_DE_TIENDA, TU_TOKEN y LA_URL_DEL_WEBHOOK (la que obtuviste en el Paso 2):
    1. Registrar evento de Orden Creada (order/created): 
    curl -X POST "https://api.tiendanube.com/v1/TU_ID_DE_TIENDA/webhooks" \
-H "Authentication: bearer TU_TOKEN" \
-H "Content-Type: application/json" \
-d '{"event": "order/created", "url": "LA_URL_DEL_WEBHOOK"}'
    2. Registrar evento de Orden Pagada (order/paid): 
    curl -X POST "https://api.tiendanube.com/v1/TU_ID_DE_TIENDA/webhooks" \
-H "Authentication: bearer TU_TOKEN" \
-H "Content-Type: application/json" \
-d '{"event": "order/paid", "url": "LA_URL_DEL_WEBHOOK"}'
    Si la consola te responde con un JSON que contiene un "id", ¡el webhook se registró exitosamente!
  5. 5. ¡Verifica Todo!
    Una vez completados todos los pasos, es crucial realizar una verificación exhaustiva.
    Realiza una venta de prueba en tu Tiendanube, verifica que aparezca en tu sección de "Pedidos Web" en el POS, que el stock se haya descontado, y que, si configuraste la auto-facturación, se haya generado la factura en ARCA/AFIP.
    No te saltes este paso. Una pequeña prueba puede ahorrarte muchos dolores de cabeza.
¿Cómo funciona la sincronización? (Reglas de Oro)
  • De POS a Tiendanube (Productos): Funciona automáticamente mediante un "robot" interno de Firebase. Cuando creas o editas un producto en el POS, el cambio viaja instantáneamente a la nube. No necesitas webhooks para esto.
  • De Tiendanube a POS (Ventas): Utiliza los Webhooks que configuraste en el paso 4. Cuando alguien compra en tu web, Tiendanube "toca el timbre" del POS enviando los datos para descontar el stock y avisarte de la nueva orden.
  • ⚠️ REGLA DE ORO: El POS 2025 es tu "fuente de la verdad". NUNCA edites precios, stock o nombres directamente en el panel administrador de Tiendanube. Esos cambios NO viajarán de vuelta al POS, y la próxima vez que guardes el producto en el POS, sobrescribirá lo que habías cambiado en la web. Todo debe ser administrado siempre desde tu sistema POS.

¡Felicitaciones! Tu POS 2025 estará listo para operar en un entorno de producción, con todas las integraciones y automatizaciones activas.
